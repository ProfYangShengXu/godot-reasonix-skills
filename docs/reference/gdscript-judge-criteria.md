# GDScript LLM Judge 评分标准

> 评判角色在判分 GDScript 代码时使用的评分维度。
> 每项 0-10 分，满分 100，≥80 为 PASS。

## 评分维度

### 1. 语法正确性（0-10）
- Godot 4.x 语法兼容（无 `onready` 等旧语法）
- `@export` 标注正确
- 类型标注正确（`var x: int` 而非 `var x`）
- `String`, `float`, `int` 类型兼容 Godot 4.x

### 2. 节点引用安全（0-10）
- 使用 `@onready var` 而非 `$` 在 `_ready()` 前
- `is_inside_tree()` 检查（可选但推荐）
- 信号连接使用类型安全的方式

### 3. 物理与碰撞（0-10）
- `move_and_slide()` 正确使用
- Layer/Mask 配置正确
- `_physics_process(delta)` 中使用 `delta` 而非硬编码

### 4. 生命周期管理（0-10）
- `_ready()` 初始化，`_process()` 更新，`_exit_tree()` 清理
- `queue_free()` 正确使用
- Tween 动画完成后清理

### 5. 信号与通信（0-10）
- 信号定义有参数文档
- Connect 使用 Callable 而非字符串
- 不跨场景直接引用（用信号/ autoload 替代）

### 6. 错误处理（0-10）
- `push_error()` / `push_warning()` 用于异常
- 空引用检查（`if node:` 而非 `if node != null:`）
- 资源加载失败降级

### 7. 性能意识（0-10）
- `_process()` 中不做耗时操作
- 不使用 `get_node()` 每帧调用
- 对象池化（频繁创建销毁的场景）

### 8. 代码组织（0-10）
- 函数 ≤ 30 行
- `@export` 变量放在文件顶部
- 私有函数用 `_` 前缀

### 9. 日志与可调试性（0-10）
- 关键路径有 `print("LOG:...")` 输出
- 错误有完整上下文
- 可选：使用 `log_utils.gd` 结构化日志

### 10. PRD 对齐（0-10）
- 接口签名与 PRD 一致
- 验收标准可运行

---

## 分档

| 分数 | 判定 | 含义 |
|------|------|------|
| 90-100 | 优秀 | 生产级代码，可直接发布 |
| 80-89 | 良好 | 少量小问题，建议修复后合并 |
| 70-79 | 及格 | 功能正确但有多项改进空间 |
| < 70 | 不及格 | 需重构后重新评判 |

## 使用

评判角色读取 `.gd` 文件后按以上 10 维逐项评分，
输出 `score: N/100` + 每个扣分维度的理由和行号引用。
